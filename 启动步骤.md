# 59Music 启动步骤

本文档指引你从零开始搭建并运行 59Music 项目。

---

## 环境要求

| 工具 | 最低版本 | 说明 |
|------|----------|------|
| **Node.js** | 18+ | 前端开发与构建 |
| **CMake** | 3.16+ | C++ 后端构建系统 |
| **Visual Studio 2022** | 17+ | Windows 下 C++ 编译器（MSVC），需安装"使用 C++ 的桌面开发"工作负载 |

> 本项目已内置 miniaudio 单头文件，并通过 CMake FetchContent 自动下载 TagLib 和 SQLite 源码，无需额外安装 vcpkg。

---

## 第一步：克隆仓库

```bash
git clone <仓库地址> 59Music
cd 59Music
```

---

## 第二步：安装前端依赖

```bash
cd frontend
npm install
cd ..
```

---

## 第三步：构建后端

```bash
cd backend
mkdir -p build
cd build
cmake ..
cmake --build .
```

首次配置时 CMake 会自动下载 TagLib 和 SQLite 源码，请保持网络畅通。

编译产物会生成在 `backend/build/Debug/` 目录下：

| 产物 | 说明 |
|------|------|
| `59music.exe` | 主服务程序（HTTP + WebSocket + 播放引擎） |
| `59music_server.exe` | 与 59music.exe 完全相同（历史遗留，忽略） |
| `queue_test.exe` | PlayQueue 逻辑测试 |
| `audio_test.exe` | AudioDecoder 解码测试 |
| `player_test.exe` | PlayerEngine 播放测试 |
| `gen_test_wav.exe` | 生成测试用 WAV 文件 |

---

## 第四步：启动后端服务

```bash
cd backend/build/Debug
./59music.exe
```

启动后终端输出：

```
59Music Server
  HTTP:      http://localhost:1059/api/v1
  WebSocket: ws://localhost:1059/ws
[Server] 监听端口: 1059
```

后端默认监听 **1059** 端口。如需指定其他端口：

```bash
./59music.exe 8888
```

按 `Ctrl+C` 退出服务。

---

## 第五步：启动前端开发服务器

打开另一个终端：

```bash
cd frontend
npm run dev
```

Vite 启动后输出：

```
  VITE v8.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

浏览器打开 `http://localhost:5173` 即可使用。

> 前端开发服务器已将 `/api`、`/ws` 请求代理到后端 `localhost:1059`，无需额外配置。

---

## 开发流程

```
┌──────────────────────┐     ┌──────────────────┐
│      终端 1           │     │    终端 2         │
│                      │     │                  │
│ cd backend/build/Debug│    │ cd frontend      │
│ ./59music.exe        │     │ npm run dev      │
│                      │     │                  │
│ 后端 API 服务         │     │ 浏览器打开        │
│ 端口 1059            │     │ localhost:5173    │
└──────────────────────┘     └──────────────────┘
```

- **修改后端代码**：在 `backend/build/` 中执行 `cmake --build .` 增量编译，重启 `backend/build/Debug/59music.exe`
- **修改前端代码**：Vite 自动热更新，无需手动刷新浏览器
- **前后端联调**：前端通过 Vite 代理访问后端，开发体验与生产环境一致

## 运行测试

后端提供独立测试程序，用于验证播放引擎各模块：

```bash
cd backend/build/Debug

# PlayQueue 逻辑测试（无需音频文件）
./queue_test

# 生成测试用 WAV 文件（15 秒 440Hz 正弦波）
./gen_test_wav

# AudioDecoder 解码测试
./audio_test test_sine.wav

# PlayerEngine 播放测试（会出声约 10 秒）
./player_test test_sine.wav
```

## 项目架构概览

```
59Music/
├── frontend/                  # Vue 3 前端
│   └── src/
│       ├── api/               # API 请求封装（player / library / queue / playlist / background）
│       ├── stores/            # Pinia 状态管理（player / library / playlist / background）
│       ├── views/             # 页面组件（音乐库 / 歌单 / 设置 / 背景管理）
│       ├── composables/       # 组合式函数（快捷键 / WebSocket / 多选 / 拖拽排序）
│       ├── utils/             # 工具函数（时间格式化）
│       └── router/            # Vue Router 路由定义
│
├── backend/                   # C++ 后端
│   └── src/
│       ├── main.cpp           # 入口：启动 HTTP 服务，监听 1059 端口
│       ├── server/            # HTTP 服务层（REST API + WebSocket + CORS）
│       ├── database/          # SQLite3 数据库层（歌曲 / 歌单 / 背景表）
│       ├── scanner/           # 音乐库扫描（文件扫描 + TagLib 元数据解析）
│       ├── player/            # 播放引擎（miniaudio 音频播放 + 播放队列）
│       └── models/            # 共享数据结构（SongInfo / PlayerState / PlayMode / BackgroundInfo）
│
└── docs/                      # 项目文档（API 文档 / 开发计划）
```

## 常见问题

### Q: CMake 配置时下载 TagLib 失败？

确保网络可访问 GitHub。如在国内网络环境下载缓慢，可配置代理后删除 `build` 目录重试。

### Q: 编译时提示找不到 Windows SDK？

确保 Visual Studio 2022 安装了"使用 C++ 的桌面开发"工作负载，安装后重启终端。

### Q: 前端 `npm install` 报错？

```bash
cd frontend
rm -rf node_modules
npm cache clean --force
npm install
```

### Q: 前端页面打开但无法连接后端？

确认后端 `59music.exe` 已启动，终端应显示"监听端口: 1059"。检查是否有防火墙阻止本地 1059 端口。

### Q: 启动后端时报"端口已被占用"？

1059 端口可能被其他程序占用，指定其他端口启动：

```bash
cd backend/build/Debug
./59music.exe 8899
```

然后修改 `frontend/vite.config.js` 中的代理目标端口为对应值。
