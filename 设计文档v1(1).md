# 59Music 设计文档

> C++ 后端 + Vue 前端，本地音乐播放器，第一期：本地播放、播放队列、播放模式、音乐库。

---

## 1. 音乐播放器原理

### 1.1 核心流程

```
音频文件 → 解码 → PCM原始数据 → 声卡输出 → 听到声音
```

### 1.2 音频文件

音频文件（如 `.mp3`、`.flac`）存储的是**压缩后的音频数据**。MP3 通过删掉人耳听不到的声音来压缩体积，FLAC 是无损压缩。这些数据不能直接送给声卡播放。

### 1.3 解码

解码将压缩数据还原为 **PCM（脉冲编码调制）** 原始数据——本质上是一个巨大的 float 数组，每秒 44100 个采样点，每个点代表该瞬间声波的振幅。

```
.mp3 文件 (5MB)  →  解码  →  PCM数据 (30MB+)
```

### 1.4 送入声卡播放

C++ 端使用 miniaudio 这样的库：
- 向操作系统申请音频设备
- 设定采样率（44100Hz）、声道数（2）、格式（float32）
- 提供一个**回调函数**，声卡每次需要数据时调用，获取下一段 PCM 数据
- 将解码好的数据喂给回调，即可播放

```
miniaudio 回调："给我下 1024 个采样点"
     → 从解码器中取 1024 个采样点
     → 返回给 miniaudio
     → 送入声卡播放
```

### 1.5 播放控制

| 操作 | 实现方式 |
|------|---------|
| 播放/暂停 | 控制是否往回调里填数据 |
| 拖动进度条 | 修改解码器的读取位置（seek），下次回调从新位置取数据 |
| 切歌 | 关闭当前解码器，打开新文件的解码器 |
| 音量调节 | 将 PCM 数据乘以系数（如 0.5 表示一半音量），或调整系统音量 |

### 1.6 前后端配合

```
前端 Vue                      后端 C++
   │                            │
   │  POST /play {songId:1}     │
   │ ──────────────────────────►│  PlayerEngine.play(songId)
   │                            │    → AudioDecoder.open("song.mp3")
   │                            │    → miniaudio 开始播放
   │   WebSocket: progress      │
   │ ◄──────────────────────────│  每秒推送当前秒数
   │                            │
   │  POST /seek {position:60}  │
   │ ──────────────────────────►│  AudioDecoder.seek(60秒)
```

前端只发指令、收状态，真正干活的（解码、音频输出）都在 C++ 后端。

## 2. 项目概述

### 2.1 项目定位

59Music 是一款基于 C++ 后端 + Vue 前端的本地音乐播放器。用户通过浏览器界面操作，实际音频解码与播放由 C++ 后端完成。

### 2.2 运行形态

C++ 后端作为本地进程运行，启动 HTTP/WebSocket 服务；Vue 前端在浏览器中打开，通过 API 与后端通信。

```
用户浏览器 ──HTTP/WS──▶ C++ 后端进程 ──miniaudio──▶ 声卡
```

### 2.3 V1 功能范围

第一版只做基础功能，目标是**能用、稳定**。

| 模块 | 功能 |
|------|------|
| 音频播放 | 播放/暂停/停止、上一首/下一首、进度跳转、音量调节 |
| 格式支持 | MP3、FLAC、WAV |
| 播放队列 | 添加歌曲、移除歌曲、清空队列、拖拽排序 |
| 播放模式 | 顺序播放、单曲循环、列表循环、随机播放 |
| 音乐库 | 添加文件夹扫描入库、歌曲列表展示（歌名/歌手/专辑/时长）、搜索 |

## 3. 技术选型

### 3.1 后端 (C++)

| 用途 | 选型 | 说明 |
|------|------|------|
| 音频解码+播放 | miniaudio | 单头文件、轻量、跨平台、支持 MP3/FLAC/WAV |
| 元数据解析 | TagLib | C++ 原生，读取歌名/歌手/专辑/时长 |

| HTTP+WebSocket | cpp-httplib | header-only、API 简洁、内置 WS 支持 |
| JSON | nlohmann/json | header-only、语法简洁 |
| 构建 | CMake + vcpkg | vcpkg 统一管理 C++ 依赖 |

### 3.2 前端

| 用途 | 选型 | 说明 |

| 框架 | Vue 3 + TypeScript | 前端 UI 框架 |

## 4. 架构设计

### 4.1 整体架构

```
浏览器 (Vue 前端)
    │  HTTP REST API (指令)
    │  WebSocket    (状态推送)
    ▼
C++ 后端进程
    ├── HttpServer       → 接收 HTTP 请求，分发给各模块
    ├── WsServer         → 往前端推送播放进度/状态
    ├── PlayerEngine     → 音频解码 + 播放控制
    ├── PlayQueue        → 播放队列 + 播放模式
    ├── LibraryManager   → 文件扫描 + 元数据解析 + 入库查询
    └── Database         → SQLite 读写
```

### 4.2 后端模块职责

| 模块 | 职责 |
|------|------|
| **HttpServer** | 注册 API 路由，解析请求参数，调用业务模块，返回 JSON |
| **WsServer** | 管理 WebSocket 连接，监听 PlayerEngine 状态变化并推送给前端 |
| **PlayerEngine** | 调用 miniaudio 解码和输出，提供 play/pause/stop/seek/volume 接口 |
| **PlayQueue** | 维护歌曲 ID 列表和当前位置，根据播放模式决定下一首 |
| **LibraryManager** | 递归扫描文件夹、TagLib 解析标签、SQLite 读写歌曲信息 |
| **Database** | 建表、连接管理、提供 DAO 接口 |

### 4.3 数据结构

```cpp
// 播放模式枚举
enum class PlayMode {
    Sequential,   // 顺序播放：队列播完即停
    SingleLoop,   // 单曲循环：当前曲播完从头开始
    ListLoop,     // 列表循环：队列播完回到第一首继续
    Shuffle       // 随机播放：打乱队列顺序
};

// 歌曲信息结构体 — 对应数据库 songs 表
struct SongInfo {
    int64_t id;          // 数据库主键，自增
    std::string filePath;// 音频文件绝对路径，例如 "D:/Music/稻香.mp3"
    std::string title;   // 歌名，从 ID3 标签读取，失败则回退到文件名
    std::string artist;  // 歌手，从 ID3 标签读取，失败则填"未知歌手"
    std::string album;   // 专辑，从 ID3 标签读取，失败则填"未知专辑"
    int duration;        // 歌曲时长，单位秒，由解码器获取
    int64_t addedTime;   // 加入音乐库的时间戳（Unix 秒）
};

// 播放状态结构体 — 通过 WebSocket 推送给前端
struct PlayerState {
    bool isPlaying;             // true: 播放中, false: 暂停/停止
    SongInfo currentSong;       // 当前播放的歌曲完整信息
    int currentPosition;        // 当前播放进度，单位秒
    int duration;               // 当前歌曲总时长，单位秒
    int volume;                 // 当前音量，范围 0-100
    bool muted;                 // 是否静音
    PlayMode mode;              // 当前播放模式
};
```

前端自行设计页面布局和组件结构，通过文档中约定的 API 和 WebSocket 协议与后端通信。

## 5. 数据库设计

使用 SQLite3，数据库文件存放在程序运行目录下的 `data/59music.db`。

### 5.1 songs（歌曲表）

存储音乐库中所有已索引歌曲的元数据，`file_path` 唯一防止重复入库。

```sql
CREATE TABLE IF NOT EXISTS songs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 主键自增
    file_path   TEXT    NOT NULL UNIQUE,            -- 文件绝对路径，去重依据
    title       TEXT    NOT NULL DEFAULT '未知歌曲',  -- 歌名，解析失败用默认值
    artist      TEXT    NOT NULL DEFAULT '未知歌手',  -- 歌手
    album       TEXT    NOT NULL DEFAULT '未知专辑',  -- 专辑
    duration    INTEGER NOT NULL DEFAULT 0,         -- 时长（秒）
    added_time  INTEGER NOT NULL                    -- 加入时间戳（Unix 秒）
);

-- 搜索和排序时的常用索引
CREATE INDEX IF NOT EXISTS idx_songs_title  ON songs(title);
CREATE INDEX IF NOT EXISTS idx_songs_artist ON songs(artist);
CREATE INDEX IF NOT EXISTS idx_songs_album  ON songs(album);
```

### 5.2 play_queue（播放队列表）

持久化当前播放队列，应用重启后自动恢复。`sort_order` 决定播放顺序。

```sql
CREATE TABLE IF NOT EXISTS play_queue (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 主键自增
    song_id     INTEGER NOT NULL,                   -- 关联 songs.id
    sort_order  INTEGER NOT NULL,                   -- 播放顺序，0 为第一首
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_queue_order ON play_queue(sort_order);
```

### 5.3 settings（设置表）

键值对存储应用配置，重启后恢复用户偏好。

```sql
CREATE TABLE IF NOT EXISTS settings (
    key   TEXT PRIMARY KEY,  -- 配置项名称
    value TEXT NOT NULL       -- 配置项值（字符串存储，读取时转换类型）
);
```

**预置配置项：**

| key | 默认值 | 说明 |
|-----|--------|------|
| volume | 80 | 音量 0-100 |
| play_mode | sequential | 播放模式：sequential / single_loop / list_loop / shuffle |
| last_song_id | -1 | 上次播放的歌曲 ID，-1 表示无 |
| last_position | 0 | 上次播放位置（秒） |

## 6. API 设计

### 6.1 约定

- 基础路径：`http://localhost:8080/api/v1`
- 成功响应：`{"success": true, "data": {...}}`
- 失败响应：`{"success": false, "msg": "错误描述"}`
- 少量参数用 URL 查询参数，批量数据用 JSON 请求体

### 6.2 播放器控制

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/player/play?songId=1` | 播放指定歌曲 |
| POST | `/player/toggle` | 播放/暂停切换 |
| POST | `/player/next` | 下一首 |
| POST | `/player/previous` | 上一首 |
| POST | `/player/seek?position=60` | 跳转进度（秒） |
| POST | `/player/volume?volume=80` | 设置音量（0-100） |
| POST | `/player/mode?mode=shuffle` | 设置播放模式，取值：sequential / single_loop / list_loop / shuffle |
| GET | `/player/state` | 获取当前播放状态 |

**GET /player/state 响应示例：**

```json
{
  "success": true,
  "data": {
    "isPlaying": true,
    "currentSong": {
      "id": 1, "title": "稻香", "artist": "周杰伦",
      "album": "魔杰座", "duration": 223,
      "filePath": "D:/Music/稻香.mp3", "addedTime": 1716883200
    },
    "currentPosition": 60,
    "duration": 223,
    "volume": 80,
    "muted": false,
    "mode": "sequential"
  }
}
```

### 6.3 播放队列

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/queue` | 获取整个队列 |
| POST | `/queue/add` | 添加歌曲，请求体 `{"songIds": [1, 2, 3]}` |
| DELETE | `/queue/{index}` | 移除队列中第 index 首 |
| DELETE | `/queue` | 清空队列 |
| POST | `/queue/reorder?from=3&to=1` | 将第 from 首移到第 to 位 |

### 6.4 音乐库

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/library/scan?folder=D:/Music` | 扫描文件夹入库，返回新增数量 |
| GET | `/library/songs?page=1&size=50&keyword=稻香` | 分页查询，keyword 可选，搜索歌名/歌手/专辑 |
| DELETE | `/library/songs/{id}` | 从库中移除歌曲（不删除原文件） |
| GET | `/library/stats` | 统计信息：歌曲总数、总时长等 |

### 6.5 WebSocket 推送

连接地址：`ws://localhost:8080/ws`

前端建连后，服务端主动推送以下消息：

| 消息类型 | 触发时机 | 数据 |
|---------|---------|------|
| `player_state` | 播放/暂停/切歌/连上时 | 完整 PlayerState |
| `progress` | 每秒一次 | `{"position": 61, "duration": 240}` |
| `library_updated` | 扫描文件夹完成 | `{"addedCount": 15}` |
| `queue_changed` | 队列增删改 | 完整队列列表 |

## 7. 任务分工

四人分工如下：

| 成员 | 模块 | 职责 | 涉及技术 |
|------|------|------|---------|
| P1 | 播放引擎 | PlayerEngine + AudioDecoder + PlayQueue + 播放模式 | miniaudio |
| P2 | 音乐库 | LibraryManager + FileScanner + MetadataParser + Database/DAO | TagLib, SQLite |
| P3 | 服务整合 | HttpServer + WsServer + main.cpp + 构建配置 | cpp-httplib, nlohmann/json, CMake |
| P4 | 前端 | Vue3 页面 + 组件 + WebSocket 对接 | Vue3, Element Plus, Pinia |

各模块对接，实现完整功能。

Bug 修复、异常处理、体验优化。
